üìã What You Need

To use this application, you need:

  • Plaid Account - Create one at plaid.com
  • API Keys - Get your Client ID and Secret from the Plaid dashboard
  • Access Token - Generated by completing the Plaid Link flow

üöÄ Setup Steps

Step 1: Get Plaid Credentials

  1. Go to plaid.com and create an account
  2. Log in to the Plaid Dashboard
  3. Navigate to Team Settings ‚Üí Keys & Credentials
  4. Copy your Client ID and Secret

Step 2: Get Access Token

Run the setup script locally to get an access token:

cd credit-history-app
python scripts/setup_plaid_token.py

This will guide you through the Plaid Link flow to authorize your financial accounts.

Step 3: Configure Environment Variables

Set these in your Render environment or .env file:

Required Variables:

PLAID_CLIENT_ID Your Plaid Client ID from the dashboard
PLAID_SECRET Your Plaid Secret from the dashboard
PLAID_ACCESS_TOKEN Generated by setup_plaid_token.py script
PLAID_ENV Either "sandbox" or "production" (default: sandbox)
üí° Tip: For Render, add these as environment variables in your service settings.

‚ùì Frequently Asked Questions

What is Plaid?

Plaid is a secure financial data aggregator. It allows you to connect your bank accounts and credit cards to this application without sharing your passwords directly.

Is my data secure?

Yes. Plaid uses bank-level encryption and security. Your credentials are never stored; only the access token is used to fetch your data.

Can I use this with a different bank?

Yes! Plaid supports 12,000+ financial institutions including all major US banks and credit unions.

Do I need a Render account?

Only if you want to deploy this application. You can run it locally without Render.

How do I run this locally?

git clone https://github.com/odanree/credit-history-app
cd credit-history-app
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your credentials
python -m src.app
